6: Middleware is a function that sits between the request and response cycle. It can modify the request or response, perform checks (like 
authentication or validation), or log details before passing control to the next function or route.

JWT is a compact and secure way to transmit information between client and server as a digitally signed token.
Commonly used for authentication and authorization in modern web apps.

How JWT Works (Step-by-Step)

User logs in with username/password.

Server verifies credentials and generates a JWT (signed with a secret key).

Server sends this token to the client (usually stored in localStorage, sessionStorage, or cookies).

For future requests, the client sends this token in the Authorization header (Bearer token).

The server verifies the token‚Äôs signature to ensure it‚Äôs valid.

If valid, the request proceeds; otherwise, the server returns a 401 Unauthorized.

11. What are the advantages and disadvantages of using Tailwind Css

A: ‚ÄúTailwind CSS is great for building fast, consistent, and responsive UIs with minimal custom CSS. It‚Äôs highly customizable and works well 
with modern frameworks like React. However, it can make your code look cluttered and has a slight learning curve in the beginning.‚Äù

Advantages of Tailwind CSS

Utility-first approach:
Tailwind provides low-level utility classes (like flex, mt-4, text-center) that let you build complex designs directly in your markup without 
writing custom CSS files.

Highly customizable:
You can easily configure your design system in the tailwind.config.js ‚Äî setting colors, spacing, fonts, breakpoints, etc., to match your 
project‚Äôs branding.

Faster development:
Since you don‚Äôt switch between CSS and JSX files often, UI building becomes much quicker, especially for prototypes and component-based 
frameworks like React or Next.js.

Automatic responsive design:
Tailwind‚Äôs responsive utilities (like md:w-1/2 or lg:flex) make it straightforward to create mobile-friendly layouts.

Consistent styling:
Utility classes enforce a consistent design system across the app, reducing style drift and ‚Äúmagic numbers.‚Äù

No naming conflicts:
You don‚Äôt have to worry about class name collisions or creating complicated naming conventions (like BEM).

Tree-shaking support:
Tailwind removes unused CSS during build time, which keeps the final CSS bundle small and optimized.

Disadvantages of Tailwind CSS

Readability concerns:
JSX can look cluttered because of too many class names, especially in complex components.

<div className="flex justify-between items-center p-4 bg-gray-100 rounded-lg shadow-md">


Learning curve:
Beginners need some time to memorize commonly used classes and breakpoints.

Inline style feel:
Some developers feel it mixes style and structure, making markup less clean or harder to scan.

Less semantic separation:
Because styling happens in JSX instead of CSS files, design and logic can get intertwined if not managed properly.

Over-customization can get complex:
Deep custom configurations in tailwind.config.js can become hard to maintain if the project grows too large.
